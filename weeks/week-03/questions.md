# Вопросы для самопроверки

1. В чем разница между Forward Proxy (обычный прокси) и Reverse Proxy (обратный прокси)?

Forward Proxy (обычный прокси):

- Находится перед клиентами
- Клиенты знают о прокси и настраивают его в браузере

Reverse Proxy (обратный прокси):

- Находится перед серверами
- Клиенты НЕ знают о прокси 

2. Что такое Upstream в терминологии Nginx?

Upstream — это группа серверов (бэкендов), на которые Nginx перенаправляет запросы.

3. Зачем скрывать реальные IP-адреса сервисов от клиента?

- Безопасность: Злоумышленник не знает где реально находятся сервисы
- Гибкость: Можно менять адреса сервисов без уведомления клиентов
- Единая точка входа: Клиенту нужно знать только адрес gateway
- Изоляция: Внутренняя сеть скрыта от внешнего мира

4. Какие еще задачи, кроме маршрутизации, может выполнять API Gateway (назовите хотя бы 2)?

- Аутентификация и авторизация: Проверка JWT токенов, API ключей
- Логирование и мониторинг: Сбор метрик со всех сервисов
- Балансировка нагрузки: Распределение запросов между экземплярами

5. Что произойдет, если Gateway упадет? Как можно от этого защититься?

Что произойдет:

- Все сервисы станут недоступны для клиентов
- Клиенты будут получать ошибки подключения

Как защититься:

- Запустить несколько экземпляров gateway
- Настроить автоматическое восстановление/перезапуск
