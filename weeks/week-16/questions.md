# Вопросы для самопроверки

1. Что такое OWASP Top 10? Назовите 3 любые уязвимости оттуда.

OWASP Top 10 - регулярно обновляемый список 10 самых критичных уязвимостей, рисков безопасности.

Топ риски 2025:

**1.** Broken Access Control
**2.** Security Misconfiguration
**3.** Sorfwate Supply Chain Failures

2. Чем Аутентификация отличается от Авторизации?

**Аутентификация** -  проверка подлинности (подтверждение личности пользователя)
**Авторизации** - проверка прав (определяет, какие данные и действия доступны после входа)

3. Что такое SQL Injection и как ORM помогают от неё защититься?

**SQL Injection** - тип уязвимости веб-приложений, при котором злоумышленник внедряет вредоносный SQL-код во входные данные, отправляемые на сервер. Манипулирует / получает доступ

4. Почему нельзя хранить секреты (API ключи, пароли БД) в коде репозитория?

Такие секреты будут доступны для всех, в открытом доступе. 

Начнем с того, что GitHub блокирует такие добавления в репозиторий.

(Keystore/Keychain) - системные хранилища ключей, операции шифрования происходят внутри него.

5. Что такое "Man-in-the-Middle" (MitM) атака и как HTTPS защищает от неё?

Man-in-the-Middle (MitM) атака - тип кибератаки, при которой злоумышленник тайно встраивается между двумя участниками обмена данными, перехватывая, читая или изменяя сообщения. 

6. Зачем нужен принцип "минимальных привилегий" (Least Privilege)?

 Необходим для защиты ИТ-инфраструктуры, предоставляя пользователям и программам только те права, которые нужны для текущей работы.
