# Вопросы для самопроверки

1. В чем главное концептуальное отличие GraphQL от REST?

Отличие во взаимодействии клиента с сервером.

REST - сервер определяет структуру и формат ответа. Он всегда неизменен. Клиент всегда получает фиксированный набор полей. Клиент не сможет сделать "выбор"

GraphQL - клиент сам выбирает какую информацию он хочет видеть. Сервер предоставляет возможность получить желаемую информацию.

2. Что такое Schema Definition Language (SDL)?

Это яхык описания схемы данных в GraphQL. SDL — это способ объявить, какие типы данных существуют, какие поля у них есть и как они связаны между собой.

3. Чем Query отличается от Mutation? Могут ли они выполняться 
параллельно?

Query: Предназначен для чтения данных. Это похож на GET-запросы.

Mutation: Предназначен для изменения данных (создание, обновление, удаление). Похоже на POST/PUT/DELETE.

Они могут выполнятся параллельно

4. Что такое "резолвер" (resolver) и кто его вызывает?

Резолвер — это функция-обработчик, которая содержит логику получения данных для конкретного поля в схеме GraphQL.

Вызывает его GraphQL-движок.

5. Как решается проблема N+1 запросов в GraphQL (dataloader)?

С помощью библиотеки DataLoader, которая позволяет обьединять запросы и загружать связанные данные за один запрос

6. Почему в GraphQL сложно реализовать кэширование на уровне HTTP (как в REST)?

В отличие от REST, где кэширование часто основано на URL-запросах, в GraphQL всё сложнее, так как один запрос может содержать несколько полей и сущностей. Поэтому эффективное кэширование требует более тонкого подхода.
