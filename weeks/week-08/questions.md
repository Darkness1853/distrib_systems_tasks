# Вопросы для самопроверки

1. Что такое Server Streaming? Приведите пример использования.

Server Streaming  - тип удаленного вызова. Где клиент отправляет один запрос на который получает множество ответов. Соединение остается открытым до завершения.

Пример: Биржа - клиент получает данные в реальном времени, сервер шлет ему данные по мере поступления/изменения

2. Чем Server Streaming отличается от просто получения большого списка (List) в одном ответе?

Server Streaming : меньше времени, меньше памяти требуется (сервер/клиент), постепенно обрабатывает, по мере поступления.

Server Streaming позволяет клиенту начать обработку данных сразу же, не ожидая полной загрузки

3. Какие метрики производительности существуют (Latency, Throughput, CPU Usage)?

***Latency*** - задержка

***Throughput*** - пропускная способность 

***CPU Usage*** - использование процессорва

***Memory Usage*** - использование памяти

***Error Rate*** - частота ошибок

4. Почему важно переиспользовать канал (Channel) в gRPC клиенте при замерах производительности?

Потому что создание при переиспользовании канала требуются заново подключаться/присоединятся (TCP,TLS)

Если не переиспользовать, то каждый запрос будет создавать новое соединение - задержка. Больше времени займет именно на установку соединения, а не на обработку запроса

"Важно мерить честно. Использовать одну машину. Исключить сетевые задержки (localhost)."

6. Что такое "Head-of-Line Blocking" и как HTTP/2 (и gRPC) решает эту проблему?

 "Head-of-Line Blocking" -  когда один медленный запрос блокирует все остальные
 
 HTTP/2 (и gRPC) использует мультиплексирование, когда несколько потоков в одном соединении

 Это решает проблему. Быстро выполняет запросы, имеет одно соединение на все запросы
