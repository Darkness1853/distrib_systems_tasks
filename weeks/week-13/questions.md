# Вопросы для самопроверки

1. Зачем нужен Helm? Какую проблему он решает?

Helm автоматизирует создание и управление Kubernetes-манифестами через шаблоны, избавляя от создания множества одинаковых стуктур микросервисов

2. Как работает приоритет значений (Values)? Кто победит: `values.yaml` или флаг `--set`?

флаг `--set` имеет больший приоритет, т.к. сначала values.yaml, затем родительские чарты, потом файлы -f, и  --set

1. Что такое функция `include` и `_helpers.tpl`? Зачем они нужны?

_helpers.tpl - файл с переиспользуемыми шаблонами, функция include вставляет их в другие шаблоны, чтобы не было дублирования

4. Почему в шаблонах Helm так важно следить за отступами и зачем нужна функция `indent` или `nindent`?

Формат файлов yaml требует от пользователя строго следить за отступами. 

Функция nindent добавляет нужное количество пробелов к каждой строке чтобы сохранить структуру.

5. Как откатить неудачный релиз с помощью Helm?

Откатить неудачный релиз можно командой

```bash
helm rollback <release-name> <revision-number>
```

6. Можно ли использовать Helm только для генерации YAML, без установки в кластер?

Да, можно этой командой 
```bash
helm template <name> ./chart -f values.yaml 
```
