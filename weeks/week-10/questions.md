# Вопросы для самопроверки

1. Что такое слой в Docker и почему порядок команд в Dockerfile важен для кэширования?

Слой — результат каждой команды (RUN, COPY, ADD).  Порядок важен т.к. при изменении слоя все последующие пересобираются. Правило такое - Часто меняющееся код ставить в конец, редко меняющееся - в начало.

2. В чем суть Multi-stage build? Какие проблемы он решает?

Разделяет сборку на этапы: в одном образе компилируем/ставим зависимости, в финальный копируем только нужное. Решает проблему больших образов.

3. Зачем нужен файл `.dockerignore`?

Чтобы исключить лишние файлы из сборки т.к. .git, __pycache__, .venv, тесты.

4. Чем отличаются инструкции `CMD` и `ENTRYPOINT`?

CMD - исп. для предоставления аргументов по умолчанию при запуске контейнера, если ENTRYPOINT не задан

ENTRYPOINT - исп. для определения осн. команды, которая будет выполнятся при запуске контейнера

5. Почему рекомендуется использовать `COPY` вместо `ADD`?

COPY - позволяет копировать в образ. Экономя место в образе. Прост в использовании.

ADD - Делает тоже что и COPY, но также умеет распаковывать архивы и скачивать по URL

6. Почему приложение в контейнере должно слушать `0.0.0.0`, а не `127.0.0.1`?

0.0.0.0 — слушает на всех интерфейсах. 

127.0.0.1 — только внутри контейнера. Не будет принимать запросы от хоста
